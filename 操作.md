# 🛠 项目操作指南：从零开始搭建运行

> 本文档按步骤说明如何从硬件准备到云端部署，完整运行本项目。

## 📦 硬件准备

### 硬件清单

| 组件 | 数量 | 说明 |
|------|------|------|
| i.MX6ULL 开发板 | 1 | 百问网 100ask_imx6ull |
| USB 摄像头 | 1 | 支持 UVC 协议，MJPEG 输出 |
| DHT11 温湿度模块 | 1 | GPIO115 |
| PIR 人体红外模块 | 1 | GPIO116 |
| MQ-2 烟雾传感器 | 1 | GPIO117（数字输出） |
| 继电器模块（风扇） | 1 | GPIO118 |
| 继电器模块（LED灯） | 1 | GPIO119 |
| 光敏电阻模块 | 1 | ADC 通道3 |
| 5V 电源适配器 | 1 | 给开发板供电 |
| USB 转串口模块 | 1 | 调试用 |


## 🔧 编译

### 主机环境要求

```bash
# 工具链
arm-buildroot-linux-gnueabihf-gcc 7.5.0

# 系统
Ubuntu 18.04/20.04 x86_64
```

### 编译步骤

```bash
# 1. 共享库（cJSON + 看门狗）
cd shared_lib && make clean && make
cd ..

# 2. RPC Server（端口1234 + HTTP 8080）
cd lesson5/rpc_server && make clean && make
cd ../..

# 3. MQTT Bridge（智能网关）
cd lesson6 && make clean && make
cd ..
```

### 编译产物

| 文件 | 大小 | 说明 |
|------|------|------|
| `lesson5/rpc_server/rpc_server` | ~200KB | RPC + HTTP 服务 |
| `lesson6/mqtt_bridge` | ~500KB | MQTT 网关 |

## 🚀 部署到开发板

### 方式一：一键部署

```bash
./deploy.sh board <开发板IP>
```

### 方式二：手动部署

```bash
# 上传二进制
scp lesson5/rpc_server/rpc_server root@<IP>:/usr/local/bin/
scp lesson6/mqtt_bridge root@<IP>:/usr/local/bin/
scp config.json root@<IP>:/root/

# 上传 Web 前端
scp lesson5/rpc_server/www/index.html root@<IP>:/usr/local/bin/www/

# 配置 MQTT 凭据
scp .env root@<IP>:/root/
ssh root@<IP> "chmod 600 /root/.env"

# 安装系统服务
scp lesson5/rpc_server/rpc_server.service root@<IP>:/etc/systemd/system/
scp lesson6/mqtt_bridge.service root@<IP>:/etc/systemd/system/
```

## ▶️ 启动服务

### 方式一：手动启动（调试用）

```bash
# SSH 登录开发板
ssh root@<IP>

# 启动 RPC Server（前台运行）
/usr/local/bin/rpc_server

# 另开终端，启动 MQTT Bridge（前台运行）
ssh root@<IP>
/usr/local/bin/mqtt_bridge
```

### 方式二：系统服务（生产用）

```bash
ssh root@<IP>
systemctl enable rpc_server
systemctl enable mqtt_bridge
systemctl start rpc_server
systemctl start mqtt_bridge

# 查看状态
systemctl status rpc_server
systemctl status mqtt_bridge
```

## ☁️ 云端部署

### 阿里云 ECS 要求

| 项目 | 说明 |
|------|------|
| 系统 | Ubuntu 20.04+ |
| Python | 3.8+ |
| 开放端口 | 1883(MQTT), 9090(图片), 3000(Grafana) |

### 启动云端服务

```bash
# 上传脚本到阿里云
./deploy.sh cloud

# 或者手动部署
scp cloud/mqtt_to_influxdb.py root@<云IP>:/opt/iot_mqtt_to_influx/
scp cloud/requirements.txt root@<云IP>:/opt/iot_mqtt_to_influx/

# 安装依赖并启动
ssh root@<云IP>
cd /opt/iot_mqtt_to_influx
pip3 install -r requirements.txt
nohup python3 -u mqtt_to_influxdb.py > mqtt_to_influxdb.log 2>&1 &

# 启动 Grafana
cd /opt/iot && docker-compose up -d
```

### 验证云端

```bash
# 检查 MQTT 桥接日志
tail -f /opt/iot_mqtt_to_influx/mqtt_to_influxdb.log

# 检查图片服务器
curl http://<云IP>:9090/  # 预期 404，说明服务在运行

# 访问 Grafana
http://<云IP>:3000  # 默认账号 admin/admin123
```

## 📊 数据流速查

```
上行遥测:
  传感器 → HAL → RPC → MQTT Bridge → MQTT → 云端 InfluxDB → Grafana

下行控制:
  云端/Web 指令 → MQTT/HTTP → MQTT Bridge/RPC → HAL → GPIO → 执行器

图片上传:
  烟雾报警 → 摄像头抓拍 → HTTP优先 → 云端存图 → 钉钉告警
                        → MQTT兜底 → 云端解码 → 钉钉告警

Web 管理:
  浏览器 → HTTP :8080 → RPC → HAL → 传感器/执行器
```

## 🐛 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| Web 界面打不开 | rpc_server 没运行 | `systemctl start rpc_server` |
| 跳匝咔嗒声 | 控制逻辑冲突 | 检查 smoke_fan_until>0 守卫 |
| 摄像头拍照黑屏 | AE/AWB 未稳定 | 已在代码中丢弃前5帧 |
| 图片传不到钉钉 | HTTP 不通或超时 | 检查阿里云 9090 端口 |
| MQTT 连不上 | 凭据错误或网络不通 | 检查 .env 配置 |
