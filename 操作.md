## 三、主机端单元测试

测试不依赖硬件的函数（config、error处理等），**在主机用 gcc 编译运行**：

```bash
cd lesson6 && gcc -DTEST_MAIN -o test test_cases.c error.c config.c -lm -I. && ./test && rm -f test
```

测试内容：
- 错误码字符串和值 (`test_error_*`)
- 配置加载默认值 (`test_config_defaults`)
- 配置JSON解析 (`test_config_parse`)
- MQTT配置 (`test_config_mqtt`)
- GPIO配置 (`test_config_gpio`)
- 阈值范围 (`test_config_threshold_range`)
- 上报间隔 (`test_report_interval_range`)

## 四、部署到开发板

### 方式一：scp 单文件传

```bash
# 编译
cd lesson5/rpc_server && make clean && make
cd ../../lesson6 && make clean && make

# 传文件到板子
scp lesson5/rpc_server/rpc_server root@<板子IP>:/root/
scp lesson6/mqtt_bridge root@<板子IP>:/root/

# 传配置文件
scp config.json root@<板子IP>:/root/
```

### 上传 .env 和 service 文件

```bash
# 环境变量配置
scp .env root@<板子IP>:/root/.env
ssh root@<板子IP> "chmod 600 /root/.env"

# systemd服务文件
scp lesson6/mqtt_bridge.service root@<板子IP>:/etc/systemd/system/
scp lesson5/rpc_server/rpc_server.service root@<板子IP>:/etc/systemd/system/
```

## 五、板端启动服务

### 方式A：systemd 开机自启

```bash
ssh root@<板子IP>

# 先启动 rpc_server
systemctl daemon-reload
systemctl enable --now rpc_server
systemctl status rpc_server

# 再启动 mqtt_bridge
systemctl enable --now mqtt_bridge
systemctl status mqtt_bridge
```

### 方式B：手动启动（调试用）

```bash
ssh root@<板子IP>

# 先启动 rpc_server（开一个终端）
cd /root && ./rpc_server

# 再启动 mqtt_bridge（另一个终端，先设环境变量）
export MQTT_HOST=8.140.232.52
export MQTT_PORT=1883
export MQTT_USERNAME=zhuxiangbo
export MQTT_PASSWORD=13979831637Zhu@
export RPC_HOST=127.0.0.1
export RPC_PORT=1234
cd /root && ./mqtt_bridge
```

> Ctrl+C 停止前台进程。

## 六、验证传感器

### 1. Web API 验证（最方便）

在开发板上执行：

```bash
# 传感器数据
curl http://localhost:8080/api/sensors
# 返回示例: {"pir":0,"light":1234,"smoke_digital":1,"temp":26,"humi":45,"relay":1,"relay2":0}

# 传感器状态
curl http://localhost:8080/api/sensor_status

# 设备信息
curl http://localhost:8080/api/device_info

# 系统状态
curl http://localhost:8080/api/system
```

或者从外部访问：`http://<板子IP>:8080/api/sensors`

### 2. 控制执行器

```bash
# 开灯（继电器2）
curl -X POST http://localhost:8080/api/relay/2/control \
  -H "Content-Type: application/json" -d '{"state":1}'

# 关灯
curl -X POST http://localhost:8080/api/relay/2/control \
  -H "Content-Type: application/json" -d '{"state":0}'

# 开风扇（继电器1）
curl -X POST http://localhost:8080/api/relay/1/control \
  -H "Content-Type: application/json" -d '{"state":1}'

# LED控制
curl -X POST http://localhost:8080/api/led/control \
  -H "Content-Type: application/json" -d '{"state":1}'
```

### 3. 摄像头抓拍

```bash
curl http://localhost:8080/api/camera/capture
# 返回: {"success":1,"size":xxxxx,"file":"camera_capture.jpg","timestamp":xxxxx}

# 查看抓拍列表
curl http://localhost:8080/api/camera/list
```

### 4. 直接通过 HAL 验证（开发板）

```bash
# 直接查看GPIO状态
cat /sys/class/gpio/gpio116/value    # PIR
cat /sys/class/gpio/gpio117/value    # 烟雾
cat /sys/class/gpio/gpio118/value    # 继电器1（风扇）
cat /sys/class/gpio/gpio119/value    # 继电器2（灯）

# 光敏ADC
cat /sys/bus/iio/devices/iio:device0/in_voltage3_raw
```

## 七、MQTT远程控制

### 从云服务器订阅遥测数据

```bash
mosquitto_sub -h localhost -t "device/telemetry" \
  -u zhuxiangbo -P "13979831637Zhu@"
```

### 从云服务器远程控制设备

```bash
# 控制LED开
mosquitto_pub -h localhost -t "device/control" \
  -m '{"method":"led_control","params":[1]}' \
  -u zhuxiangbo -P "13979831637Zhu@"

# 控制灯（继电器2）开
mosquitto_pub -h localhost -t "device/control" \
  -m '{"method":"relay2_control","params":[1]}' \
  -u zhuxiangbo -P "13979831637Zhu@"

# 读取温湿度
mosquitto_pub -h localhost -t "device/control" \
  -m '{"method":"dht11_read","params":[]}' \
  -u zhuxiangbo -P "13979831637Zhu@"

# 远程抓拍
mosquitto_pub -h localhost -t "device/control" \
  -m '{"method":"camera_capture","params":[]}' \
  -u zhuxiangbo -P "13979831637Zhu@"
```

### 查看响应

```bash
mosquitto_sub -h localhost -t "device/response" \
  -u zhuxiangbo -P "13979831637Zhu@"
```

## 九、日志查看

### 通过 Web API

```bash
# 查看所有日志（最近100条）
curl http://localhost:8080/api/logs

# 按级别筛选
curl "http://localhost:8080/api/logs?level=error"

# 关键词过滤
curl "http://localhost:8080/api/logs?filter=smoke"
```

### 通过 SSH

```bash
# 查看 rpc_server 日志
ssh root@<板子IP> "journalctl -u rpc_server -n 50 -f"

# 查看 mqtt_bridge 日志
ssh root@<板子IP> "journalctl -u mqtt_bridge -n 50 -f"
```
